Linux F.A.Q

Can I run GBM on Linux?

  • As of the latest v0.96 pre-release, GBM will run on Linux using Mono. You can detect and backup Linux native games and games running in Wine.

What's required to run GBM on Linux?

  • GBM requires a few prerequisites to be installed. Some of these may be pre-installed on your distribution.
    • Mono Framework
      • Visit mono-project.com for the best way to install this on your distro.
    • Mono Visual Basic .NET support and WinForms
      • The mono-vbnc package and it's dependencies will cover this on Debain based distros.
    • SQLite3 library
      • The libsqlite3-0 package on Debain based distros.
    • 7-Zip
      • The p7zip-full package on Debain based distros.

How do I install and run GBM on Linux?

  • Download the latest package from Releases and extract it to a folder. Navigate to the folder in a terminal and use the command mono GBM.exe to run it.

Are there any limitations while running in Linux?

  • Unfortunately there's a few:
    • The tray icon and menu will not function in Linux and are disabled. You can minimize GBM to the task bar instead.
    • All automatic file/folder searches are disabled in Linux due to being unreliable. This feature will be enabled again once I get around to fixing it in Mono.
    • You may encounter slight visual glitches with the interface, mostly with listbox controls. These glitches are related to Mono and your video driver or version.
    • GBM may not exit cleanly if a critical error occurs, the process will need to be killed.

How do I configure GBM to detect Wine games?

  • You don't need to do anything special. Simply set the process to the game's executable name (without the extension), exactly like you do while using GBM on Windows.
  • Please note that while GBM will display the game running as "wine-preloader", it still knows game's actual working directory and will properly backup a relative path configuration.

Can GBM share the same backup folder between Linux and Windows?

  • Yes it can, you can play a game on Linux and restore your saved games to continue on your Windows machine and vice versa.
  • There are some important tips you should know before doing this.
    1. Linux and Windows will very rarely share the same game configurations. Disable the "Sync game list data with backup folder" option in Settings, then use seperate configurations on each OS with matching names.
    2. The restore path will be displayed incorrectly when restoring saved games between operating systems, but GBM will automatically detect and fix the path when you do a restore, just choose Yes to the confirmation!

I'm getting an error message when I run GBM!

  • The following are some common errors and the reason:
    • The entry point method could not be loaded.
      • This indicates that Mono Visual Basic or WinForms support is not installed.
    • 7-Zip was not found. The application cannot continue.
      • This indicates that 7-Zip is not installed or that 7za is not located in /usr/bin.
    • Other exceptions or crashes on startup.
      • Any other error message is probably related to SQLite3 not being installed or detected by Mono.

I'm having other problems running GBM in Linux!

  • GBM in Linux is very much experimental at this time. Please report any issues!

Game Launcher F.A.Q

Why would I use Game Backup Monitor to my launch games?

  • Game Backup Monitor is always available in the system tray, making it a quick way to launch a game.
  • After a game has been detected at least once, GBM usually has all the information required to launch it.
  • This makes GBM a lightweight and simple game launcher, but advanced customization options are available if required.
  • This feature is fully optional and completely invisible by default if you don't want to use it.

Why can't I launch games with Game Backup Monitor?

  • The game launching feature is disabled by default, it must be enabled in the General section of File -> Settings.
  • This feature relies on the optional "Session Tracking" feature, it must be enabled as well.

How do I launch games with Game Backup Monitor?

  • Launch any of the last five recently played games by right-clicking the system tray icon and choosing the game you'd like to play.
  • Use the "Quick Launcher" to find and launch any configured game. It's accessed via the tray menu or using "Ctrl + L" on the main window.
  • Launch any game directly from the Game Manager using the "Launch Options" button.

Why do I always get an error about missing information when trying to launch my games?

  • Most games will need to be detected for at least one session in v1.2.4 before GBM has enough information to automatically launch it. You can also manually provide the information required to launch a game in the Game Manager.

Why can't I launch a game even after it's been detected once?

  • In rare cases, required information such as the "Game Path" can't be detected. The reason is usually displayed in the log. When this happens you can manually set the missing information in the Game Manager.

Why do some of my games crash when I launch them from Game Backup Monitor?

  • This can be a problem with certain games purchased on services such as Steam or Epic Game Store.
  • Some games will simply refuse to work properly when launched directly from their executable and they need to launched with their respective store client. This can be DRM related or simply a bug.
  • GBM supports launching games through compatible store clients, but the game must be manually configured to do so in the Game Manager.

I need to launch certain games as Administrator or with compatability settings. Is this supported?

  • Yes. Any game executable flagged to "Run As Administrator" will request elevation as it normally does and will use any compatability settings.
  • In addition, any game launched while GBM is running as Administrator will automatically launch as Administrator. However this is not recommended, and GBM will ask for confirmation before every launch.

Is this feature supported on Linux?

  • Yes, with one limitation. Any Windows game you are attempting to launch from GBM in Linux will need to be manually configured to launch via an external launcher or script.
  • A future release may support automatically building the launch command using detected Wine/Proton information, but it will not be supported initially.

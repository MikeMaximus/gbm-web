Contribution Guide [Official Game List]

What do I need to contribute to an official game list?

  • You need a GitHub account and basic knowledge on how to work with Git and GitHub.
  • The latest release version of Game Backup Monitor.

Are there any standards to follow?

  • Refrain from adding tags that do not already exist in the official game list.
  • Try to avoid including files that contain machine specific settings (such as graphics settings) in your backup configurations. When this isn't possible, make a note of it in the comments field.
  • Game configurations that use a variable save location will be rejected, such as any game that only saves to steam cloud folder.

How do I contribute?

  1. Fork a copy of the gbm-web project.
  2. Import all the game configurations on the current official list (Windows or Linux) you're contributing to into GBM. You can maintain a seperate list for contributing if you wish.
  3. Add the game configurations you want to contribute, you may also edit existing entries if they are out of date or broken.
  4. Make sure to test each configuration you've added or changed.
  5. The new configurations must be tagged with #Official and the platforms you've tested. #GOG and/or #Steam for example.
  6. When ready to submit your changes, export your list from the Game Manager to GBM_Official.xml (or GBM_Official_Linux.xml).
  7. The export should be filtered by tag #Official and sorted by Name.
  8. The following line needs to be added to the XML file with a text editor at line 2:
    • <?xml-stylesheet type="text/xsl" href="GBM_Official.xsl"?>
    • Replace GBM_Official.xsl with GBM_Official_Linux.xsl for Linux
  9. Add the completed XML file to your fork of gbm-web, commit and push the changes.
  10. Create a pull request.
  11. Once I approve the pull request, your changes will be added to the official list.

How do I maintain a seperate list for contributing?

  • GBM stores it's settings and local configuration list in C:\Users\*username*\AppData\Local\gbm (Windows) or /home/*username*/.local/share/gbm (Linux).
  • You can maintain multiple copies of this folder by renaming it as needed.
  • For example, rename your current folder from "gbm" to "gbm_personal". Then run GBM to create a fresh list for contributing. To switch back, rename "gbm" to "gbm_official" and rename "gbm_personal" to "gbm".
  • You can maintain an unlimited amount of lists in this manner, but don't rename folders while GBM is running.
